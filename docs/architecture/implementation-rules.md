# Implementation Rules

Development is sprint-gated.

- Implement one sprint per iteration.
- Keep generated code around 500 lines unless explicitly approved.
- Leave the project runnable after every sprint.
- Include tests and documentation updates in every sprint.
- Stop after each sprint and wait for approval.
- Do not refactor unrelated modules.
- Do not change public interfaces unless strictly necessary.
- Keep changes small and logically grouped.

Milestone 2 must not start until Milestone 1 is reviewed and approved.

## Sprint Audit Format

Each sprint summary must report:

- sprint summary
- technical review scores
- problems found
- improvements made
- technical debt
- recommendations
